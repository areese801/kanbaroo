# Changelog

All notable changes to Kanberoo are recorded here. This project follows [Semantic Versioning](https://semver.org/) and [Keep a Changelog](https://keepachangelog.com/).

## [Unreleased]

### Fixed
- TUI editor flow (`c` comment, `e` edit description, `n` new story) no longer crashes on vim exit. The default editor runner previously wrapped `with app.suspend():` inside `asyncio.to_thread`, entering Textual's suspend context on a worker thread and leaving the terminal state inconsistent when the subprocess unwound. The runner now enters `app.suspend()` on the event loop and offloads only the blocking `subprocess.run` to a thread, which is what Textual's driver attach/detach logic assumes, and calls `app.refresh()` on the way out so the first frame after vim exits repaints cleanly.
- TUI low-priority card badge color changed from bright blue (low contrast on most terminal themes) to sage green (`#7faa3a`) so it remains distinct from the card background without reading as "urgent" or stealing attention from medium/high priority.
- Spelling: "initialised" is now "initialized" in the `kb init` Rich panel (user-visible) and the TUI's internal "AsyncApiClient not initialized yet" error message. No behavior change.
- TUI navigation bindings (`h`/`j`/`k`/`l` and `q`) on the board, workspace list, epic list, and epic detail screens now carry `priority=True` so they fire even when a descendant widget (story card, data table) holds focus. Previously Vim-style navigation and back/quit were silently swallowed in real terminal sessions. Bindings that legitimately yield to focused widgets (`m`, `enter`, `n`, `/`, `a`, `E`, `?`, `r`) still do. Workspace list `q` binding now targets `app.quit` explicitly so the Screen-to-App action fallback works under every Textual dispatch path.
- Case-insensitive by-key lookups in the service layer. `services.stories.get_story_by_human_id`, `services.epics.get_epic_by_human_id`, and `services.workspaces.get_workspace_by_key` now match with `func.upper` on both sides so every client (REST, CLI, TUI, MCP, future) can address `KAN-3`, `kan-3`, or `Kan-3` interchangeably. Fixed at the service layer so no client has to remember to normalise.

### Changed
- TUI board and epic-detail column navigation (`h`/`l`/`left`/`right`) now skip empty columns. With cards in Backlog, Todo, and Done but empty In-Progress and In-Review, pressing `l` from Todo lands on Done in a single keystroke instead of walking through each empty column. A no-op when every column to the requested side is empty.
- TUI actor badge on story cards, comment headers, story-detail audit rows, and the global audit feed now renders as an emoji (`👤` human, `🤖` claude, `⚙️` system) on terminals that support glyph rendering, with a single-letter (`h`/`c`/`s`) fallback when `$TERM` is `dumb` or contains `linux`. The fallback matches the prior one-character label so no display regresses.
- TUI workspace list `q` binding now pushes a small confirm modal instead of quitting outright; `y`/`enter` confirms, `n`/`escape` dismisses. A second `q` within 500ms of the first skips the modal and exits directly for power users who know they want out. Back-navigation `q` on non-terminal screens (board, story detail, epic list, epic detail, audit feed) is unchanged since going back is cheap and reversible.
- TUI story-detail screen gains keyboard tab navigation: `1`-`5` jump to Description/Comments/Linkages/Tags/Audit by index; `[`/`]`, `tab`/`shift+tab` walk through the tabs one at a time with wrap-around. Mouse clicks on the tab bar still work.
- TUI tag picker modal now focuses its `SelectionList` on open so arrow keys, `j`/`k`, `space`, and `enter` work immediately; same fix applied to the link picker's `OptionList` so `j`/`k` move the cursor between link types.
- TUI screen title bar (Textual Header) now shows a descriptive per-screen `sub_title` ("Workspaces", "KAN - Board", "KAN - KAN-3", "KAN - Epics", "KAN - Epic KAN-1", "Audit", "Search") so the current location is obvious at a glance.
- TUI `W` / `E` / `A` are now app-level priority bindings that reset the screen stack and navigate fresh. `W` returns to the workspace list from any depth; `E` opens the epic list for the workspace in context (on the workspace list itself it preserves today's highlighted-row behavior, and on story detail it fetches the workspace via REST before navigating); `A` pops to the workspace list and pushes the global audit feed. Makes deep-stack navigation predictable without walking back screen by screen.
- `resolve_workspace` in the CLI no longer walks the workspace list on 404; mirrors the MCP resolver now that `GET /workspaces/by-key/{key}` exists. A dashless reference routes straight to `/workspaces/by-key/{key}`; a hyphenated reference tries the direct UUID endpoint first and falls back to by-key for the rare hyphenated-key case. The two CLI tests that exercised the previous list-scan fallback now assert the by-key path.
- `docs/spec.md` section 7.2 gains a "Default workspace" paragraph documenting the flag → env → config resolution order and the new `kb workspace use` / `kb workspace current` commands.

Story detail only knows the workspace id; the app's handler fetches the full body via `GET /workspaces/{id}` when the deeper stack has no workspace dict available.

- Default workspace across the CLI. `--workspace` on `kb story create/list`, `kb epic create/list`, `kb tag create/list`, and `kb export` is now optional; the effective workspace resolves from the flag first, then `$KANBEROO_WORKSPACE`, then the new optional `default_workspace` key in `$KANBEROO_CONFIG_DIR/config.toml`. When none of the three supplies a value the CLI exits 1 with a Rich-styled hint pointing at all three paths. `kb workspace use KEY` validates the key against the live server (via `/workspaces/by-key/{key}`) and rewrites `config.toml` to persist the default; a failed validation leaves the file untouched. `kb workspace current` reports the effective default plus the source supplying it (`flag` / `env` / `config` / `unset`). `kanberoo_cli.config.CliConfig` gains an optional `default_workspace: str | None`; `write_default_workspace` performs the on-disk rewrite.
- Tests: service-layer case-insensitive lookup tests for stories, epics, and workspaces; an API-level `GET /api/v1/stories/by-key/{kan-3}` test confirming the round-trip. CLI tests for `kb workspace use` (rewrites config, rejects bogus keys without touching the file), `kb workspace current` (unset/env/config branches), `kb story list` resolution order (flag > env > config and the clean error when none is set), and the new by-key path on `kb workspace show`. TUI tests for `h`/`l`/`j`/`k` under a focused story card, `q` with a focused descendant on the workspace list and board, plus sanity checks that non-priority bindings (`m`, `r`) still fire.

- Workspace export for milestone 16 (spec section 2.4) in `kanberoo-core` and `kanberoo-api`. `services.export.export_workspace` builds a `tar.gz` archive in memory containing a `schema_version.json` manifest (alembic revision, workspace identity, created-at), `tables/*.parquet` per table scoped to the target workspace (via pyarrow), and a self-contained `kanberoo.db` SQLite copy with the workspace's rows. The SQLite copy uses a fresh temp database so `api_tokens` can be dropped before serialising; credential material never ships with an export. Linkages that touch a workspace story or epic are included on both ends (cross-workspace linkages are allowed per spec section 10 Q2). `GET /api/v1/workspaces/{id}/export` streams the archive with `Content-Type: application/gzip` and a `KAN-export-<iso>.tar.gz` disposition; `?include_deleted=true` opts in to soft-deleted rows (off by default). Adds `pyarrow>=15` to `kanberoo-core` runtime deps.
- Audit read surface for spec section 4.2 in `kanberoo-core` and `kanberoo-api`. `services.audit.list_audit` returns newest-first events with optional filters (`entity_type`, `entity_id`, `actor_type`, `actor_id`, `since`) and opaque cursor pagination wrapping `(occurred_at, id)` so ties on `occurred_at` are stable. `list_audit_for_entity` is a convenience wrapper behind the dedicated path. New router `GET /api/v1/audit` plus `GET /api/v1/audit/entity/{entity_type}/{entity_id}` return `{items, next_cursor}` envelopes. `AuditEventRead.diff` now parses the stored JSON string into a structured `{"before": ..., "after": ...}` dict so clients do not double-decode. The TUI's existing audit feed and story-detail audit tab, and the MCP `get_audit_trail` tool, all start working end-to-end once this endpoint is live.
- `GET /api/v1/workspaces/by-key/{key}` for parity with `/stories/by-key` and `/epics/by-key`. Backed by `services.workspaces.get_workspace_by_key` (raises `NotFoundError` on miss; hides soft-deleted rows unless `include_deleted=true`). Lets clients and MCP resolve `KAN` to a full workspace without a list scan.

Both screens subscribe to the WS fan-out and refetch on `epic.*` or `story.*` events that touch them. New `OpenEpicList` and `EpicSelected` messages route the flow through the app.

- MCP server for milestone 15 (spec section 6) in `kanberoo-mcp`. The new `kanberoo-mcp` console script is a thin translator between the MCP tool protocol and the Kanberoo REST API; it runs over stdio and authenticates to the API with its own bearer token. Seventeen tools exposed (spec section 6.2): `list_workspaces`, `get_workspace`, `list_epics`, `create_epic`, `update_epic`, `list_stories`, `get_story` (folds comments and linkages into the response), `create_story`, `update_story`, `transition_story_state`, `comment_on_story`, `link_stories`, `unlink_stories`, `list_tags`, `add_tag_to_story`, `remove_tag_from_story`, `get_audit_trail`. Tool names and descriptions are crafted for an outer AI agent rather than cloned from endpoint names; `update_story` deliberately omits state so state changes route through `transition_story_state` and are attributed on the server side.
- `kanberoo_mcp.config.resolve_config` reads configuration from four sources in order: `--api-url`/`--token` CLI flags, `--token-env NAME` indirection, `$KANBEROO_MCP_TOKEN` or `$KANBEROO_TOKEN` env vars, and the shared `$KANBEROO_CONFIG_DIR/config.toml`. Missing config exits 1 with a stderr message listing the resolution order.
- `kanberoo_mcp.client.McpApiClient` is a sync `httpx.Client` wrapper mirroring the CLI client but independent from it so the two front ends can evolve separately. Translates the canonical error envelope into typed `McpApiRequestError` and exposes `fetch_etag`/`patch_with_etag`/`post_with_etag` helpers for optimistic-concurrency endpoints. The `transport=` kwarg lets tests swap in `httpx.MockTransport`.
- `kanberoo_mcp.resolver` hides human-id and key resolution: workspace refs resolve via `GET /workspaces/{key}` with a list-scan fallback (no `/workspaces/by-key` endpoint exists yet), stories and epics go through `/by-key/{human_id}`, and tags resolve by name within a workspace.
- `kanberoo_mcp.concurrency.with_retry_on_412` wraps every mutating tool handler so a single `412 Precondition Failed` self-heals by refetching the ETag and retrying once. A second consecutive 412 surfaces a clean "entity changed underneath us" message rather than looping.
- Claude-actor detection: on startup the server performs a best-effort hash-match against `GET /api/v1/tokens` to determine the resolved token's `actor_type` and logs a warning (not a gate) when it is not `claude`. An MCP deployment using a human token still works; mutations are just attributed as human in the audit log.
- Test harness under `packages/kanberoo-mcp/tests/`: `httpx.MockTransport`-backed `MockApi` plus 46 tests covering the full registry surface (every expected name present, server's `list_tools` matches), config resolution order, every tool's happy path and argument shaping, human-id resolution (KAN-N triggers `/stories/by-key/KAN-N`, UUID triggers `/stories/{id}`), the 412 single-retry and double-412 error paths, tag-by-name resolution, server-level error translation (`McpApiRequestError` becomes an `isError=true` tool result, not a server crash), and the invalid-state-transition surfacing from the REST layer.
- TUI surfaces for milestones 13 and 14 (spec section 8) in `kanberoo-tui`: story detail, fuzzy search, global audit feed, shared help overlay, and editor-integrated workflows. `StoryDetailScreen` renders five tabs (Description/Comments/Linkages/Tags/Audit) over a Textual `TabbedContent`, subscribes to WS updates for its story id, tolerates a `404` from the per-entity audit endpoint (reserved for cage K) by showing a "not yet available" state instead of failing the screen, and binds `e`/`c`/`m`/`t`/`L`/`r`/`?` to description edit (PATCH with If-Match), add comment (POST), move mode, tag picker, link picker, refresh, and help. `SearchScreen` is a full-screen fuzzy search overlay opened with `/` from the workspace list or board; it builds a client-side index on mount (walking every workspace's stories via REST), ranks with `difflib.SequenceMatcher` over weighted human-id/title/description fields with prefix and substring bonuses, re-ranks on every keystroke, and pushes `StoryDetailScreen` on `enter`. `AuditFeedScreen` (opened with `a` from the workspace list) renders `GET /api/v1/audit` rows in a `DataTable`, prepends live WebSocket events to the top, and tolerates a `404` from `/audit` cleanly with a placeholder while still populating the table from subsequent WS events.
- `kanberoo_tui.editor.edit_markdown`: async `$EDITOR` helper that writes `initial` to a temp file, drives the editor via `App.suspend` inside a worker thread (so the event loop can process queued WS frames while the user edits), reads the file back, and returns `None` when the buffer is unchanged. A `SuspendNotSupported` fallback path makes the helper work under Textual's headless driver used by `run_test`. The `runner=` kwarg is a typed injection point so tests swap in a callable that rewrites the temp file without touching a real subprocess.
- Board view keybinding `n` creates a new story: the editor opens on a `# Title (replace this line)\n\n# Description below\n\n` template; the first non-empty line becomes the title, anything after becomes the markdown description, and leaving the placeholder unchanged aborts with a flash so a stray keypress never creates a junk story. `enter` on the board now pushes the real story detail screen (replacing the cage H placeholder), and `/` posts the new `OpenSearch` message.

New `FakeEditor` fixture drives the editor flow without ever launching a real subprocess.
- TUI scaffold and board view for milestones 11 and 12 (spec section 8) in `kanberoo-tui`. New `kanberoo-tui` console script launches a Textual app that reads `$KANBEROO_CONFIG_DIR/config.toml` (same shape the CLI uses, with `$KANBEROO_API_URL` / `$KANBEROO_TOKEN` overrides) and routes the user from a workspace list into a kanban board for the selected workspace.
- `kanberoo_tui.client.AsyncApiClient`: async `httpx`-based REST client mirroring the CLI's `ApiClient` shape (bearer auth, canonical error envelope translation, ETag/If-Match helper for `POST /stories/{id}/transition`). Accepts `transport=` so tests drive it with `httpx.MockTransport`.
- `kanberoo_tui.ws.EventSubscriber`: async WebSocket subscriber that connects to `ws://.../api/v1/events?token=...`, filters out the `{"type":"ping"}` keepalive frames, and auto-reconnects with exponential backoff. Dependency-injected `connector` callable lets tests script frame sequences without a real socket.
- Workspace list screen (`WorkspaceListScreen`): landing view that fetches every workspace, adds story and epic counts (one extra request per endpoint per workspace, bounded by single-user scale), and renders a `DataTable`. Keybindings: `j`/`k` to move the cursor, `enter`/`l`/`→` to open the board, `r` to refresh, `?` for the help panel, `q` to quit. `workspace.*` WebSocket events trigger a refetch.

Each story card shows the human id, a color-coded priority badge (none/low=blue/medium=yellow/high=red), a one-character actor badge for the last state transition (`h`/`c`/`s`), and optional tag chips. Keybindings: `h`/`l`/`←`/`→` between columns, `j`/`k`/`↓`/`↑` between cards, `m` enters move mode where the next key (`b`/`t`/`p`/`r`/`d`) transitions the focused card via `POST /stories/{id}/transition` with `If-Match`, `esc` cancels move mode, `r` refreshes, `q` returns to the workspace list. `enter` and `/` flash "cage I" placeholders for the story detail and search flows reserved for the next cage. `story.*` WebSocket events trigger a refetch.

- CLI surface for milestones 9 and 10 (spec section 7.2) in `kanberoo-cli`: every non-`init` command is now implemented. New sub-apps: `kb workspace list/create/show`, `kb story list/create/show/edit/move/comment/link/delete`, `kb epic list/create/show/close/reopen`, `kb tag list/create/rename/delete`, `kb token list/create/revoke`, `kb server start/stop`. New top-level commands: `kb audit <KAN-N|UUID>`, `kb export --workspace <KEY> --output <dir>`, `kb backup [--output <dir>]`. Every list/show command supports `--json` for scripting and defaults to Rich-rendered tables otherwise. Destructive commands (`story delete`, `tag delete`, `token revoke`) prompt for confirmation unless `--yes` is passed.
- `kanberoo_cli.config.load_config` helper reads `$KANBEROO_CONFIG_DIR/config.toml` (default `~/.kanberoo/config.toml`) and applies `$KANBEROO_API_URL` / `$KANBEROO_TOKEN` / `$KANBEROO_DATABASE_URL` environment overrides. Missing config exits 1 with a Rich-rendered hint pointing the user at `kb init`.
- `kanberoo_cli.client.ApiClient` is a thin `httpx.Client` wrapper that attaches the bearer token, composes `/api/v1` paths, translates the canonical error envelope into `ApiRequestError`, and provides `patch_with_etag` / `delete_with_etag` / `post_with_etag` helpers that fetch the current ETag via GET and pass it back as `If-Match`. The `transport=` kwarg lets tests swap in an `httpx.MockTransport` without monkey-patching any handler.
- `kanberoo_cli.resolvers` hides human-id resolution: story and epic refs like `KAN-123` go through `GET /stories/by-key/{human_id}` or `GET /epics/by-key/{human_id}`; workspace refs try `GET /workspaces/{key}` first and fall back to walking the list for a key match.
- `kb story edit <KAN-N>` dumps the current description to a temp `.md` file, launches `$EDITOR` (default `vim`) in the `git commit` style, and PATCHes only when the buffer changed. An unchanged buffer short-circuits with "no changes, nothing to do.".
- `kb server start/stop` shells out to `docker compose up -d` / `docker compose down` via `subprocess.run`. `$KANBEROO_COMPOSE_FILE` flows through as `-f` so CI can point at a non-default file. `--wait` on `start` polls `/api/v1/workspaces` until a 2xx or 30 seconds elapse.
- `kb backup` is local-only: parses the SQLite path out of `database_url` and copies it with `shutil.copy2` to `<output>/kanberoo-YYYYMMDD-HHMMSS.db` (default output: `$KANBEROO_CONFIG_DIR/backups`). A non-SQLite `database_url` prints a clear `SQLite only for now` warning and exits 0 without touching the filesystem.
- `kb export` streams `GET /api/v1/workspaces/{id}/export` to `<output>/<key>-<timestamp>.tar.gz`. That endpoint is cage K's responsibility and currently returns 404; the CLI surfaces a friendly "export endpoint not available yet" message on stderr and exits 1 instead of dumping a traceback. The happy-path test uses a canned tarball-shaped byte string via `MockTransport`.
- REST addition: `GET /api/v1/epics/by-key/{human_id}` mirrors the existing story-by-key endpoint so the CLI's `--epic KAN-N` flag can resolve to a UUID without walking a workspace's epic list. Backed by a new `services.epics.get_epic_by_human_id` helper and covered by a roundtrip/404 integration test in `test_epics_api.py`.
- CLI test harness under `packages/kanberoo-cli/tests/`: a `MockApi` fixture builds an `httpx.MockTransport` bound to a scripted request/response table and a `config_dir` fixture writes a temp `config.toml`. 41 new tests cover every sub-command: workspace list (table/empty/JSON)/create/show (key+UUID+404); story list/create/show/edit (no-change short circuit and real PATCH with fake editor script)/move/comment/link/delete; epic list/create/show/close/reopen; tag list/create/rename/delete; token list/create/revoke; audit happy path and not-found; backup (SQLite copy and non-SQLite warning); export (streamed tarball and 404 message); server start/stop with mocked `subprocess.run` and compose-file override.
- WebSocket event stream for milestone 8 (spec section 5): `WS /api/v1/events?token=<plaintext>` publishes live events for every service mutation. Each event carries the full common envelope from spec section 5.3 (`event_id`, `event_type`, `occurred_at`, `actor_type`, `actor_id`, `entity_type`, `entity_id`, `entity_version`, `payload`). Event types match spec section 5.4 (`workspace.created`/`.updated`/`.deleted`, epic/story/comment/tag equivalents, `story.transitioned`, `story.commented`, `story.tag_added`/`.tag_removed`, `story.linked`/`.unlinked`). Tokens are not events (consistent with the no-audit-on-tokens invariant).
- `kanberoo-core.events` in-process pub/sub bus with async-friendly subscribe/publish and bounded per-subscriber queues (drop-on-full, client refetches via REST). `services.events.publish_event` is the sync helper used by every service.
- Events are emitted only after successful transaction commit (via SQLAlchemy `after_commit` listener), so rollbacks never leak phantom events to subscribers. Rollback-no-event is covered by an explicit test.
- 30-second keepalive ping (`{"type":"ping","ts":"<iso>"}`) on the WebSocket so clients can distinguish idle-but-connected from dead connections. Ping interval is module-configurable for tests.

### Changed
- `kanberoo_mcp.resolver.resolve_workspace` simplified to use the `/workspaces/by-key/{key}` endpoint (added in the workspace-by-key change above). A dashless reference (keys like `KAN`, `ENG`) routes straight to the by-key endpoint; a UUID-shaped reference tries the direct endpoint first and falls back to by-key. The old list-scan pagination loop is gone. MCP tool tests that previously mocked `GET /workspaces/KAN` now mock `GET /workspaces/by-key/KAN`; a new test covers the UUID + by-key-fallback path for hyphenated keys.
- `pyproject.toml` excludes `packages/kanberoo-cli/tests/` from the workspace-wide mypy run because its `conftest.py` would otherwise collide with the existing `packages/kanberoo-api/tests/conftest.py` under mypy's unique-module-name rule. Both conftests are still exercised by pytest; only the mypy call ignores the cli tests tree. Existing core-tests type coverage is unaffected.

### Fixed
- Model `Enum` columns now store the lowercase `.value` (e.g. `human`) via a shared `enum_values` helper, matching the spec and the Alembic migration CHECK constraints. Before this fix, SQLAlchemy defaulted to the enum `name` (e.g. `HUMAN`), which diverged from the migration-created schema and would reject inserts on a freshly-migrated database.
